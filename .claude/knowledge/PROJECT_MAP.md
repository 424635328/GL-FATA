# Project Map

## Core paths

| Path | Role | Notes |
| --- | --- | --- |
| `GL_FATA.m` | Standalone GL-FATA entry point | Use for direct optimization calls and root smoke checks. |
| `FATA.m` | Local FATA baseline copy | Depends on root `initialization.m`. |
| `initialization.m` | Baseline population initialization | Compatibility dependency for root FATA. |
| `tests/smoke_test.m` | Fast root verification | Uses a Sphere objective with scalar and vector boundaries. |
| `third_party/FATA/` | Upstream FATA submodule | External code; do not edit in this repository. |

## CEC2022 paths

| Path | Role | Notes |
| --- | --- | --- |
| `CEC2022/RUNCEC2022_0529.m` | Recommended current comparison entry | Eight algorithms; persists results to `CEC2022_Data.mat`. |
| `CEC2022/Analyze_0529.m` | Analysis and figure generation | Reads `CEC2022_Data.mat`. |
| `CEC2022/Run_Ablation_Study.m` | Component ablation entry | Uses five FATA variants. |
| `CEC2022/GL_FATA.m` | CEC-local GL-FATA implementation | Keep behavior aligned with root `GL_FATA.m`. |
| `CEC2022/Get_Functions_cec2022.m` | CEC objective wrapper | Calls the compiled `cec22_func` function. |
| `CEC2022/cec22_func.cpp` | MEX source | Rebuild on platforms without the included Windows binary. |
| `CEC2022/input_data22/` | CEC data files | Required by the CEC function implementation. |

## Historical and legacy paths

- `CEC2022/RunCEC2022/Run0529/` holds the images and spreadsheets used in the README.
- `CEC2022/runsCEC2022_Main.m` and `CEC2022/runsCEC2022_Main_Parallel.m` are earlier entry points with different algorithm sets and output expectations.
- Do not combine output generated by different entry points in one ranking, chart, or summary without an explicit normalization plan.
